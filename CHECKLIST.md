# 期末项目提交自查清单 (Submission Checklist)

在您点击“提交”或第 18 周上台展示前，请各小组对照本清单逐一检查。

## ✅ 1. 学术论文 (1_论文/)
- [ ] **模板规范**：是否严格使用了 `0_Guide_Materials/大学物理期刊论文模板.doc` 进行排版？
- [ ] **最终格式**：是否已将论文导出为 **PDF** 格式并命名为 `论文正文.pdf` 放在本目录下？
- [ ] **物理要素**：论文中是否包含：摘要、物理控制方程、数值算法离散化过程、稳定性分析？
- [ ] **结果分析**：图表是否清晰、有编号、有物理意义的解释？是否包含误差分析或收敛性测试？
- [ ] **图片源文件**：所有论文插图的源文件（.png/.pdf）是否已放入 `assets/` 目录？

## ✅ 2. 项目代码 (2_Code/)
- [ ] **运行指南**：`2_Code/README.md` 是否清晰写明了**如何一键运行**以重现论文中的结果？
- [ ] **逻辑解耦**：代码是否已拆分为“计算”、“分析”、“可视化”等模块？（严禁将所有代码塞进一个文件）
- [ ] **物理注释**：核心物理公式、关键循环和参数是否有清晰的中文注释？
- [ ] **依赖管理**：`requirements.txt` 是否包含了项目运行所需的所有第三方库？
- [ ] **硬编码检查**：是否已消除绝对路径？物理参数是否已在程序开头集中定义？

## ✅ 3. 现场答辩 (4_Presentation/)
- [ ] **文稿准备**：PPT（.pptx 或 .pdf）是否已上传？文件名是否符合 `[论文题目]_组号.pptx` 规范？
- [ ] **时间控制**：内容是否能在 **8 分钟**内讲完？（留出 2 分钟 Q&A）
- [ ] **演示演示**：若有动画或交互程序，是否已确认在演示环境下可稳定运行？

## ✅ 4. 团队协作与 AI 使用 (5_协作分工/)
- [ ] **分工透明**：`任务分工表.md` 是否已如实填写，且每位成员均有明确贡献？
- [ ] **AI 声明**：是否在论文或代码中明确标注了哪些部分是 AI（如 Trae, ChatGPT 等）生成的？
- [ ] **Commit 记录**：GitHub 的提交记录是否能体现出团队的协作过程？

## ✅ 5. 模拟数据 (3_Data/)
- [ ] **文件限值**：是否确保没有将单个超过 **50MB** 的数据文件上传至仓库？
- [ ] **复现说明**：若未上传数据，是否已在 README 中写清数据生成/获取的方法？

---
> **小贴士**：一份“像样”的科研成果，不仅在于结果的正确，更在于过程的可追溯与排版的专业性。祝大家答辩顺利！
